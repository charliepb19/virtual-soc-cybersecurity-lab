# virtual-soc-cybersecurity-lab
Virtual cybersecurity lab demonstrating network reconnaissance and traffic monitoring using Kali Linux and Ubuntu Server.

Virtual SOC Cybersecurity Lab

Overview

This project demonstrates the creation of a small virtual cybersecurity lab designed to simulate basic network reconnaissance and monitoring activities. The lab environment was built using virtual machines to replicate a simplified attacker–target scenario.

The objective of this project was to:

Configure a virtual network environment
Simulate a reconnaissance attack using network scanning tools
Monitor and capture network traffic generated during the attack
Analyze the results to understand how reconnaissance activity appears on a monitored system

The lab environment consists of two systems:

Attacker Machine: Kali Linux

Target / Monitoring System: Ubuntu Server

Both systems were connected through a virtual network configured using VirtualBox.

Lab Architecture

The lab uses a simple network structure where the attacker system communicates with the target server over a virtual network.

Network Configuration

System	Role	IP Address
Kali Linux	Attacker	192.168.56.102
Ubuntu Server	Target / Monitoring	192.168.56.101

Step 1 – Network Connectivity Verification

Before performing reconnaissance, network connectivity between the attacker and target machines was verified using ICMP.

Command executed on the Kali machine:

ping 192.168.56.101

This confirmed that the attacker system could communicate with the Ubuntu server across the virtual network.

Step 2 – Baseline Network Reconnaissance Scan

A reconnaissance scan was performed using Nmap from the Kali Linux system to identify open ports and services on the Ubuntu server.

Command used:

nmap -A 192.168.56.101

The scan provided information about:

Open ports
Running services
Service versions
Basic host information

This type of scan represents the reconnaissance phase of a cyberattack, where an attacker gathers information about a target system.

Step 3 – Monitoring Network Traffic

To observe the scanning activity from the defender's perspective, network traffic was monitored on the Ubuntu server using tcpdump.

Command used:

sudo tcpdump -i enp0s3 -n

While the scan was executed from Kali, tcpdump captured the packets generated by the reconnaissance activity. This demonstrates how network monitoring tools can detect suspicious behavior on a system.

Security Findings

The Nmap scan identified exposed services on the Ubuntu server. During the scan, tcpdump detected network packets originating from the Kali attacker machine. This demonstrates how reconnaissance activity can be observed through packet monitoring tools.

Key observations include:

Network scans generate identifiable packet patterns
Monitoring tools such as tcpdump can detect scanning activity
Packet inspection provides insight into attacker behavior during early stages of intrusion attempts
